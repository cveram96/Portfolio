Predicción de Cancelación de Servicio (Churn)

Comparación entre Random Forest y Perceptrón Multicapa

  • Dataset: WA_Fn-UseC_-Telco-Customer-Churn (7,043 registros)
  • Variables clave: Tenure, Contract, MonthlyCharges, InternetService
  • Mejor F1-Score: 0.634 (Random Forest con RandomUnderSampler)
  • Tecnologías: Python, scikit-learn, pandas, matplotlib, imblearn

Contexto del Proyecto

Análisis del problema de cancelación de servicio en telecomunicaciones para identificar clientes con alta probabilidad de abandonar la compañía y permitir la implementación de estrategias preventivas.

Variables Principales:

  • tenure: Número de meses como cliente
  • Contract: Tipo de contrato (Mensual, 1 año, 2 años)
  • MonthlyCharges: Cargos mensuales
  • TotalCharges: Cargos totales acumulados
  • InternetService: Tipo de servicio de internet
  • OnlineSecurity: Tiene seguridad en línea
  • TechSupport: Tiene soporte técnico
  • PaymentMethod: Método de pago

Metodología Detallada

1. Análisis Exploratorio (EDA)

  • Distribución de clases desbalanceadas (73.5% No Churn vs 26.5% Churn)
  • Análisis de asociación con Coeficiente V de Cramer para variables categóricas
  • Test de Levene y Welch's t-test para variables numéricas
  • Visualización de relaciones entre variables y Churn

2. Preprocesamiento

  • Limpieza de datos:
    • Conversión de tipos de datos
    • Imputación de valores faltantes con mediana
    • Eliminación de ID único (customerID)
  • Transformación de variables:
    • Estandarización con StandardScaler para numéricas
    • Codificación con OneHotEncoder para categóricas
  • Técnicas de balanceo:
    • SMOTE
    • RandomUnderSampler
    • ADASYN
    • SMOTEENN

3. Modelado

  • Random Forest:
    • Optimización con RandomizedSearchCV
    • Validación cruzada estratificada
    • Hiperparámetros: n_estimators, max_depth, min_samples_split
  • Perceptrón Multicapa (MLP):
    • Optimización con RandomizedSearchCV
    • Hiperparámetros: hidden_layer_sizes, activation, solver

Hallazgos Clave

  • La permanencia del cliente (tenure) es el factor más influyente
  • Los contratos a largo plazo reducen significativamente el Churn
  • El servicio de fibra óptica aumenta la probabilidad de Churn
  • El pago con cheque electrónico está asociado con mayor Churn
  • La falta de servicios adicionales (seguridad, soporte técnico) aumenta el Churn

Conclusiones y Recomendaciones

Comparación de Modelos

  • Random Forest ofrece mejor equilibrio general entre precisión y recall
  • MLP logra mayor recall pero con más falsos positivos
  • SMOTEENN y RandomUnderSampler son las mejores técnicas de balanceo

Limitaciones

  • Dataset desbalanceado (26.5% de casos positivos)
  • Variables no cumplen supuestos de normalidad
  • Posible sobreajuste con técnicas de sobremuestreo

Recomendaciones

  • Implementar Random Forest con SMOTEENN para estrategias de retención
  • Priorizar clientes con contratos mensuales y servicio de fibra óptica
  • Ofrecer incentivos para migrar a contratos a largo plazo
  • Mejorar servicios adicionales (seguridad, soporte técnico)
  • Explorar modelos alternativos como XGBoost o redes neuronales más profundas
